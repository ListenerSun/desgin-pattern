###策略模式
- 定义:定义了算法家族，分别封装起来，让它们之间可以互相转换，此模式让算法的变化
        不会影响到使用速发的用户。
- 适用场景:
    + 系统有很多类,而他们的区别仅仅在于它们的行为不同
    + 一个系统需要动态地在几种算法中选择一种
- 优点:
    + 开闭原则
    + 避免使用多重条件转移语句
    + 提高算法的保密性和安全性
- 缺点:
    + 客户端必须知道所有的策略类，并自行决定使用哪一个策略类
    + 产生很多的策略类
- 个人理解: 就是定义一个接口，接口里面有很多方法，同时有不同的实现类来实现这个接口
            一般策略类不单独使用，可以配合工厂模式，消除很多 if else 语句。
